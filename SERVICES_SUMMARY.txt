================================================================================
COMPLETE DATA FLOW
================================================================================

Lead → [ops: verify PAN] → [voice call via ElevenLabs]
  ├─ If QUALIFIED: [WhatsApp + Email checklist]
  └─ If NOT_QUALIFIED/INCOMPLETE: [End or retry]

Borrower sends documents via:
  ├─ WhatsApp (WAHA webhook → process_whatsapp_document)
  ├─ Email (Gmail push → attachments → process_whatsapp_document)
  └─ Portal (presigned S3 PUT → confirm endpoint → classify/extract_zip)

Document Processing (Celery):
  1. classify_document [ai-classification queue]
     ├─ If NEEDS_HUMAN_REVIEW (low confidence/ambiguous): End
     └─ Else: Create LogicalDoc + enqueue extraction

  2. extract_document_data [ai-extraction queue]
     └─ Call Claude Haiku → get fields → enqueue Tier1

  3. run_tier1_validation [ai-tier1 queue]
     ├─ If FAILED: Notify borrower + send status email
     ├─ If PASSED: Send status email
     └─ If all required docs passed: Enqueue Tier2

  4. run_tier2_validation [ai-tier2 queue]
     ├─ If PASSED: advance_to_underwriting → READY_FOR_UNDERWRITING
     └─ If FAILED: Mark VALIDATION_FAILED (or stay in status if FLAG_FOR_OPS)

Automated Follow-ups (Celery Beat hourly):
  - follow_up_service.check_and_send_reminders()
    └─ Days 1,3,5,7: WhatsApp + Email reminders
    └─ Day 7+: Escalate to RM via activity_feed

================================================================================
CELERY QUEUE CONFIGURATION
================================================================================

Broker: AWS SQS (or memory/eager in dev mode via CELERY_TASK_ALWAYS_EAGER)
Result backend: None (fire-and-forget; results in MongoDB)
Serialization: JSON
Timezone: Asia/Kolkata
Task acknowledgment: Late (after completion)
Worker prefetch: 1 message per worker at a time

Queues:
  ai-classification   → Classify physical files
  ai-extraction       → Extract document data
  ai-tier1            → Tier 1 validation (per-document)
  ai-tier2            → Tier 2 validation (cross-document)
  zip                 → ZIP file extraction
  voice               → Outbound calls via ElevenLabs
  whatsapp            → WhatsApp messages + doc processing
  email               → Email messages

================================================================================
MONGODB COLLECTIONS USED
================================================================================

Collections queried/updated by services:

leads
  - Used by: All services for lead data
  - Key fields: status, entity_type, mobile (encrypted), email, etc.

phys_files (PhysicalFile)
  - Used by: ai_worker, storage_service, zip_service
  - Statuses: RECEIVED → CLASSIFYING → CLASSIFIED → NEEDS_HUMAN_REVIEW
  - Or: → (create LogicalDoc) → classification data stored

logical_docs (LogicalDoc)
  - Used by: ai_worker, validation_rules
  - Statuses: READY_FOR_EXTRACTION → EXTRACTING → EXTRACTED →
              TIER1_VALIDATING → TIER1_PASSED/FAILED →
              (if all pass T1) → TIER2_VALIDATING

call_records
  - Used by: voice_service for call state + transcript storage

whatsapp_messages
  - Used by: whatsapp_service, follow_up_service for message tracking

activity_feed
  - Used by: All services for audit trail + follow_up_service for escalations

agents (Agent configurations)
  - Stores: EXTRACTION_AI, VALIDATION_AI, DOC_COLLECTION, VOICE_AI configs
  - Queried by: validation_rules service + ai_worker

CONFIGURATION REQUIREMENTS
================================================================================

MongoDB agents collection must have these agent types:

1. EXTRACTION_AI
   - classification_confidence_threshold (int, default 75)
   - extraction_fields_by_doc_type: {doc_type: [field_names]}
   - classification_prompt_additions (str)
   - extraction_prompt_additions (str)

2. VALIDATION_AI
   - tier1_rules: [{rule_id, rule_name, doc_type, enabled, custom_prompt, threshold}]
   - tier2_rules: [{rule_id, rule_name, custom_prompt}]
   - on_tier1_failure_action: NOTIFY_BORROWER_AND_CONTINUE | BLOCK_LEAD
   - on_tier2_failure_action: FLAG_FOR_OPS_REVIEW | BLOCK_LEAD
   - tier1_failure_message_templates: {rule_id: message_template}

3. DOC_COLLECTION
   - doc_checklist_by_entity_type: {entity_type: {required: [...], optional: [...]}}
   - whatsapp_checklist_template (str with {{borrower_name}} etc.)
   - reminder_whatsapp_template (str)

4. VOICE_AI
   - (Managed by ElevenLabs; just store ELEVENLABS_AGENT_ID in env)
